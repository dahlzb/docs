软件包管理器

RPM

RPM包管理员（简称RPM，全称为The RPM Package Manager）是在Linux下广泛使用的软件包管理器。RPM此名词可能是指.rpm的文件格式的软件包open in new window，也可能是指其本身的软件包管理器(RPM Package Manager)。最早由[Red Hat](https://baike.baidu.com/item/Red Hat/0?fromModule=lemma_inlink)研制，现在也由开源社区open in new window开发。RPM通常随附于Linux发行版，但也有单独将RPM作为应用软件open in new window发行的发行版（例如Gentooopen in new window）。RPM仅适用于安装用RPM来打包的软件，目前是GNU/Linuxopen in new window下软件包资源最丰富的软件包类型之一。

YUM

Yum（全称为 Yellow dog Updater, Modified）是一个在Fedoraopen in new window和RedHat以及CentOSopen in new window中的Shell前端软件包open in new window管理器。基于RPMopen in new window包管理，能够从指定的服务器自动下载RPM包并且安装，可以自动处理依赖性open in new window关系，并且一次安装所有依赖的软件包，无须繁琐地一次次下载、安装。

常用命令
# 列出所有可更新的软件清单命令
yum check-update
# 更新所有软件和系统内核
yum update
# 查找软件包
yum search <keyword>
# 列出所有可安装和已安装的软件
yum list
# 查看软件是否已安装或是否有可安装软件包
yum list <package_name>
# 安装指定软件, 加-y安装过程提示全部自动选y
yum install [-y] <package_name>
# 更新指定软件
yum update <package_name>
# 卸载指定软件
yum remove <package_name>
# 清除缓存目录下的软件包
yum clean packages
# 清除缓存目录下的 headers
yum clean headers
# 清除缓存目录下旧的 headers
yum clean oldheaders
# 清除缓存目录下的软件包及旧的 headers
yum clean all
切换国内数据源

显示已配置的源信息

yum repolist

备份系统默认源

mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup

将从URL中获取的数据写入到 -O 指定的文件中

wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

更新缓存

yum clean all
yum makecache

下载文件 Wget

查看系统有没有安装 wget

rpm -qa|grep wget

如果安装了会返回当前安装版本

wget-1.14-18.el7_6.1.x86_64

没有安装可通过以下命令安装

yum install -y wget

常用命令

# 下载文件
wget -p 保存目录 下载文件地址
# 将从URL中获取的数据写入到 -O 指定的文件中
wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

网络

查看本机电脑IP地址

ip address
ifconfig

设置固定IP

查看ens33网络配置

cd /etc/sysconfig/network-scripts
cat ifcfg-ens33

修改网络配置

vim ifcfg-ens33

在文件后添加以下内容

# 设置静态模式 static：静态IP、dhcp：动态IP
BOOTPROTO=static
# ip地址
IPADDR=192.168.165.130
# 子网掩码
NETMASK=255.255.255.0
# 网关
GATEWAY=192.168.165.2
DNS1=114.114.114.114

重启网络服务

service network restart

常见问题

Restarting network (via systemctl): Job for network.service failed because the control process exited with error code

# 关闭NetworkManager服务
systemctl stop NetworkManager
systemctl restart network.service
service network restart
